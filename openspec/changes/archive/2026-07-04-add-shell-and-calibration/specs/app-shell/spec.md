# app-shell — delta for add-shell-and-calibration

## MODIFIED Requirements

#### Requirement: Menu-bar-only application (modified)

The status-item menu SHALL contain exactly: **About Tiler**, **Settings…**, **Quit**.
(The MVP's inline permission line, gestures toggle, Diagnostics submenu and Launch at
Login move into Settings/About.) When Accessibility is missing, the status item SHALL
show a distinct alert appearance.

##### Scenario: Alert state in the menu bar
- WHEN Accessibility is not granted
- THEN the menu bar icon switches to its alert variant until the permission is granted

## ADDED Requirements

#### Requirement: App icon

Tiler.app SHALL ship a proper app icon (hand/fingers motif; owner-selected variant)
as an .icns generated by the build, plus a template-style menu bar icon (normal and
alert variants) instead of the text glyph.

#### Requirement: About window

The About window SHALL show the app name/icon, version, build timestamp (injected at
build time), and a clickable link to the GitHub repository
(https://github.com/amilabs/Tiler).

##### Scenario: Build time reflects the actual build
- WHEN the app is rebuilt and About is opened
- THEN the displayed build timestamp matches the new build, not a stale constant

#### Requirement: Startup permission flow

On launch without Accessibility, Tiler SHALL (beyond the one-shot system prompt)
show its own notice and open the path to fixing it (Settings window with the
highlighted permission row and/or the system pane).

##### Scenario: Launch without permission
- WHEN Tiler starts while Accessibility is not granted
- THEN the user sees an explicit notice and lands one click away from the system pane,
  and the app continues running normally otherwise
